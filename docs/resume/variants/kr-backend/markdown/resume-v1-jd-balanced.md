## 핵심 경력 (문제-조치-성과)

### 뽀득 | 백엔드 개발자 | 2025.04 ~

#### 1) 인수증 발행 시스템 이관

- 문제: GAS 동기 파이프라인 구조라 업체 1곳 실패 시 뒤 요청이 연쇄 중단, 월 2-3회 장애 반복
- 조치: 업체 단위 SQS 메시지로 분리하고 PDF 생성은 Python Lambda로 분산
- 성과: 발송 실패 월 2-3회 → 0건, PDF 용량 100KB → 10KB, 수기 대응 제거

#### 2) TMS 레거시 개편

- 문제: 문서/테스트 없는 코드베이스 인수 후 타임아웃 시 재시도 부재, 월 3-5회 수기 처리
- 조치: 인터페이스 추상화 + MockAdapter로 테스트 가능 구조화, Retry(최대 3회)와 Slack 알림 도입
- 성과: 수기 대응 월 3-5회 → 0회, 장애 인지 시점 "다음 날" → "즉시"

#### 3) 결제 자동화 및 PG 추상화

- 문제: API 없는 CMS로 월 3회 수기 정산/갱신, 데이터 증가에 따라 휴먼 에러 반복
- 조치: 토스페이먼츠 빌링키 정기결제 도입, PG 공통 인터페이스 + Adapter 구조 적용
- 성과: 월 3회 수기 작업 제거, 신규 PG(이니시스) 추가를 Adapter 1개 구현으로 대응

#### 4) AWS 인프라 개편

- 문제: 이관 완료 후에도 미사용 Windows EC2/RDBMS가 유지되어 비용 누수
- 조치: 인프라 자산 정리 후 메인 RDS로 통합
- 성과: 연간 약 400만 원 비용 절감

### 팀오투(카모아) | 백엔드 개발자 → 파트 리드 | 2022.07 ~ 2025.03

#### 5) 공급사 API 통합 아키텍처 (GOTAR)

- 문제: 공급사별 REST/SOAP 스펙 차이로 모놀리스 장애 전파 발생
- 조치: 공급사 래핑 서버를 독립 ECS로 분리하고 공통 스키마 패키지 정의
- 성과: 인벤토리 7개국/2천 대 → 71개국/400만 대 확장, 신규 공급사 온보딩 속도 개선

#### 6) DocumentDB 비용 최적화

- 문제: 월 140만 원 스토리지 비용이 예약 트래픽 대비 과다
- 조치: TTL Index로 1개월 초과 데이터 자동 만료, 만료 전 S3 백업
- 성과: 월 140만 원 → 12만 원(91% 절감), 데이터 유실 0건

#### 7) 렌터카+호텔 패키지 트랜잭션

- 문제: 예약/결제가 단일 트랜잭션에 묶여 신규 상품군 확장이 어려움
- 조치: 보상 트랜잭션 방식으로 도메인 분리, 웹훅 Retry + DLQ + 멱등 처리 적용
- 성과: 패키지 상품 출시, 호텔 매출 내 패키지 비중 60%
